## Description

Firefly is a crypto-native device for Ethereum founded by Ricmoo, creator of [EthersJS](https://github.com/ethers-io/ethers.js/).

  * **Human-readable transaction verification** : our goal is to support decoding calldata on-device,so you see exactly what you are signing before you sign it
  * **Open source hardware, all the way to the chip** : built on the ESP32-C3 (32-bit RISC-V, 160MHz). Schematics and PCB design files are fully public
  * **Open source firmware** : every line of code is public, readable, and verifiable
  * **Build or customise your own firmware** : fork it, flash it, make it do something new
  * **$10 target price** : hardware security accessible to everyone, not a $80-$300 luxury device



  


Firefly is an open source hardware wallet for Ethereum. The hardware designs, schematics, and firmware are all public. Built on the ESP32-C3 with a 1.3" display, BLE, and USB-C. Anyone can read it, audit it, fork it, and build on it. Just visit [github.com/firefly](https://github.com/firefly) for more info.

  


As well as using open hardware, Firefly firmware is open-source. More than that, we want you to be able to modify and customise the firmware to suit your needs. Security is not one-size-fits-all. Someone protecting a multisig treasury has different needs to someone managing their on-chain identity or signing into decentralised apps. With Firefly, you can build or modify your own firmware to match your use case. Display what matters to you. Add the context you need. The goal is that by the time you press "sign", you are confident in exactly what you are approving.

  


Hardware security should not be a luxury. That is why the target price for a Firefly Pixie is $10. The most basic Ledger costs $79 and the more recent models are $400+. For most of the world, that is a real barrier. At $10, you can put a secure signing device in the hands of every developer, every student, and every user who needs one. Or even better, work with donors to subsidise devices for users in need.

  


Funding goes toward firmware development, hardware iteration and continued free devkit distribution.

  


In case you were wondering if you can run Doom on a Firefly (because we have been asked many times by security researchers) the answer is **YES**!
