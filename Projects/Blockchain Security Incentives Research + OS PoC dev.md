## Description

**Project Description: Blockchain Security Incentives in a Changing Threat Environment**

  


**Who**

This project is led by Dr Kelsie Nabben, an established researcher in blockchain security and digital ethnography, with a demonstrated track record of contributions to the Ethereum security ecosystem. This includes _the_ book on blockchain security from a social science perspective (see: <https://manchesteruniversitypress.co.uk/9781526187086/>), and prior funded collaborations with organisations such as Shield3, zeroShadow, and BlockScience. The project builds directly on existing, publicly available research outputs and practitioner collaborations, including an ETH Rangers security grant to open access publications and an academic grant to R&D an open-source red-teaming library.

  


**What**

This project delivers an open, public-good research and tooling framework for analysing and improving blockchain security incentives within Ethereum and EVM-compatible ecosystems. It focuses on three domains:

  


  1. Incentives for protocol cooperation in incident response (e.g., fund freezes, coordination requests)
  2. Incentives shaping white-hat behaviour (e.g., responsible disclosure, rescues, fund returns)
  3. The transformation of incentives under AI-driven and automated adversarial conditions



  


Outputs will include:

  * A comparative framework for analysing how incentive mechanisms operate across public, decentralised blockchain ecosystems
  * Practitioner-oriented guidelines to inform incentive-aware incident response design
  * (open access)
  * Annotated case studies of Ethereum security incidents



  


**Why**

Security in Ethereum is not solely a technical problem but an incentive alignment challenge. As articulated by Vitalik Buterin, the goal of security is ‘to minimise divergence between user intent and system behaviour’—yet current incentive structures remain fragmented and insufficiently coordinated.

  


Ethereum security increasingly depends on the alignment of economic, social, and reputational incentives across diverse actors, including protocols, white hats, and incident responders. Meanwhile, AI-enabled adversaries and automation are accelerating exploit discovery and execution, undermining traditional mechanisms such as bug bounties.

  


This project addresses a critical gap: **there is no systematic, publicly available framework for understanding or designing incentive-compatible security coordination in Ethereum ecosystems**. By producing open, actionable qualitative knowledge, the project contributes directly to Ethereum’s security ecosystem formation and resilience as a public good.

  


**Where**

The research is grounded in Ethereum and L2 ecosystems (including EVM-compatible chains), ensuring strong ecosystem alignment. Empirical cases will focus on Ethereum-based incidents. Fieldwork will be conducted in public and semi-public coordination environments (e.g., Discord, governance forums, Telegram) and through established practitioner networks spanning industry, non-profit security collectives, and policy organisations.

  


**How**

Using a digital ethnographic methodology, the project combines participant observation, semi-structured interviews, and case study analysis of security incidents. These qualitative insights will be complemented by document analysis (e.g., post-mortems, governance proposals, an announcements) to trace how incentives are articulated and enacted in practice. The project will synthesise findings into a practical framework and set of design recommendations for incentive-aligned security coordination.

  


The final deliverable will be a **practical, written framework and set of design recommendations** that can be used by Ethereum developers, security teams, and governance actors, to be disseminated by the researcher.

  


**When**

The project will run over 12 months:(in-depth, long-form qualitative research takes time).

  


Initial phases will focus on mapping the current incentive landscape and conducting fieldwork (Months 1–6), followed by analysis and framework development (Months 7–10), and publication and ecosystem dissemination of findings through Open Access reports, practitioner briefings, and workshops (Months 11–12).

  


This research directly contributes to strengthening blockchain security by identifying actionable pathways to align incentives for cooperation, ethical behaviour, and rapid response in an increasingly automated adversarial environment.

**Public Goods Contribution & Impact**

  


All outputs will be released under open, permissive licenses and actively disseminated by the lead researcher.

This project addresses a structural gap in Ethereum security: the absence of a shared understanding of how incentives shape coordination under real-world conditions. By bridging empirical research with operational security practices, it equips the ecosystem with practical, evidence-based mechanisms to improve incentive design, and thus incident response and resilience.

  


This project contributes directly to Ethereum and L2 security as a **public good** by:

  


  * **Enhancing white-hat incentive design:** Clarifying and advancing best practice incentives for responsible disclosure, rescue operations, and post-incident fund returns in a way that is legible and actionable across the ecosystem
  * **Advancing security research and standards:** Delivering empirically grounded insights into incentive alignment, contributing to shared knowledge, emerging standards, and more coherent security practices across Ethereum
  * **Strengthening incident response coordination:** Producing openly available frameworks and guidance that improve how protocols, security teams, and ecosystem actors structure incentives during exploits
