## Description

Every Web3 project needs security review, but most can't afford it. A dedicated security engineer costs $150-300K/year. Professional triage services run $5-20K/month. The result: thousands of open-source protocols, DAOs, and public goods projects ship code with no systematic vulnerability review process.

Meanwhile, the tools that large security teams use internally — structured triage workflows, severity classification frameworks, duplicate detection, bounty management — remain locked behind enterprise contracts.

This leaves the most important part of the ecosystem — the open, permissionless, public goods layer — as the least protected.

  


## What We Built

We built and open-sourced a complete AI-powered security triage system that any project can deploy for free. It consists of three components:

### 1\. HackenProof MCP Server (Model Context Protocol)

An open-source server that connects AI assistants like Claude directly to bug bounty infrastructure. Any project team can:

  * **Review vulnerability reports against their own codebase**  — the AI reads the security report and cross-references it with the project's actual source code to validate findings, locate affected code paths, and suggest fixes
  * **Manage the full vulnerability lifecycle**  — from receiving a report to validating it, classifying severity, posting responses, and tracking resolution
  * **Run batch operations**  — process dozens of reports across multiple programs simultaneously instead of one-by-one manual review



The server exposes 25+ structured tools covering report reading, state management, severity classification, commenting, label management, attachment handling, and authenticated screenshots. It supports both API key and session-based auth, runs as a Docker container, and can be self-hosted or used via our public endpoint.

**This is not a wrapper around HackenProof's UI.** It's a structured API layer that turns AI assistants into security triage agents — giving any team the operational capability that previously required a dedicated security operations center.

Source: <https://docs.hackenproof.com/dashboard/mcp-server>

  


### 2\. Triage Skill — AI-Guided Vulnerability Assessment

A structured workflow that encodes professional triage methodology into a reusable AI skill. When a security report comes in, the skill guides the AI through:

  * **Scope validation**  — is the reported target and version actually in scope for the program?
  * **PoC verification**  — does the report include concrete proof of exploitation?
  * **Duplicate detection**  — has this vulnerability already been reported?
  * **Severity classification**  — what's the real impact, mapped to industry-standard severity levels?
  * **Decision and response**  — what's the correct triage outcome, with a professional explanation for the reporter?



This encodes years of triage experience from handling 60,000+ vulnerability reports into a tool that any team can use immediately. A solo developer running a DeFi protocol gets the same triage quality as a team with a 10-person security department.

  


### 3\. Bulk Triage Skill — Portfolio-Wide Security Monitoring

For teams managing multiple programs or protocols, the bulk triage skill:

  * Discovers all open reports across every program
  * Syncs with local codebases and blockchain explorers for validation
  * Analyzes each report and produces structured recommendations
  * Outputs everything for human review — no actions are taken without approval



This turns what would be days of manual review across multiple dashboards into a single command that produces a prioritized security status report.

Source: [github.com/hackenproof-public/skills](https://github.com/hackenproof-public/skills)

  


## Who This Is For

These tools are designed for exactly the teams that the Giveth Security Fund exists to support:

**Open-source protocol teams** — You have a codebase on GitHub and receive security reports, but don't have a dedicated triage person. Connect the MCP to Claude Code, open your repo, and ask the AI to validate incoming reports against your actual code.

**DAOs and public goods builders** — You run a bug bounty program but can't afford professional triage. The triage skill gives your contributors the same structured methodology used by professional security teams.

**Small teams managing multiple projects** — You maintain several contracts or applications and need to stay on top of security across all of them. The bulk triage skill scans everything at once and tells you where to focus.

**Any Ethereum project** — Connect the MCP, open your project in Claude Code, and say: _"Fetch new reports and check each one against our codebase."_ The AI reads the report, reads your code, and tells you what's real, where it is, and how to fix it.

  


## Why This Is a Public Good

**It's fully open source.** Skills, references, templates — all MIT licensed on GitHub. Fork it, modify it, run it yourself.

**It removes the cost barrier.** Professional security triage operations previously required either expensive staff or expensive service contracts. This makes the same capability available to any project with a GitHub repo and a Claude Code session.

**It scales with the ecosystem.** Every vulnerability report that passes through the system teaches the AI new attack patterns. A project that receives 50 reports doesn't just get 50 triage decisions — it gets a 50-pattern codebase scan that finds similar vulnerabilities the researchers didn't report.

**It compounds.** The triage skill encodes HackenProof's institutional knowledge from 60,000+ reports. Every project that uses it benefits from that accumulated experience without having to build it themselves.
