## Description

Freedom of the Press Foundation and Tor are developing [_WEBCAT_](https://securedrop.org/news/introducing-webcat-web-based-code-assurance-and-transparency/) (Web-Based Code Assurance and Transparency), a tool that protects users from tampered or malicious code when they visit websites. **This is a standard security mechanism used in app stores that is missing from the open web.** WEBCAT, which will initially be bundled with Tor Browser but is intended to inform integrity verification standards implemented in every major web browser, makes it possible for developers of security-sensitive software to digitally sign the software itself. Without WEBCAT, when a user opens a website, there's no way to verify the version of the site that's delivered to their computer hasn't been maliciously altered. WEBCAT keeps users safe by checking that the site's code matches the manifest signed by its developer. This keeps users safe before any data is processed or displayed, further strengthening the protections of encryption. 

  


WEBCAT is built with every use case in mind, which is why we will enable any application developer to use the WEBCAT plugin, first for regular websites, and eventually for anonymous websites accessible only in Tor Browser network as well. This kind of code verification is important for a wide range of users; from webmail to note-taking software, to digital wallets, and for any tool that handles sensitive communications.
