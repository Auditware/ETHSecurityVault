## Description

Assertions (assertions.eth) is a public-good smart contract that brings test-driven development to on-chain transactions. Deployed at [0xA55e4707A94Ce4Aa647517ed9aD4084e4E5D1f3F](https://etherscan.io/address/0xA55e4707A94Ce4Aa647517ed9aD4084e4E5D1f3F) on Ethereum mainnet, it provides a comprehensive suite of view-only assertion functions that can be composed into any batched transaction to enforce pre-conditions, post-conditions, and invariants, atomically on-chain.

  


## Why it matters

High-stake on-chain operations (treasury transfers, protocol upgrades, permission changes, fund recoveries) are irreversible. Off-chain simulations (like Tenderly) can be rendered inaccurate by MEV, front-running, or state changes between simulation and execution. On-chain assertions are the only mechanism that guarantees conditions hold at the exact moment of execution. They are the last line of defense.

  


## How it works

Assertion calls are batched alongside your actual operations in a DAO proposal, a Safe transaction batch (using Transaction Builder), or an EIP-7702 batched transaction. At execution time, each assertion verifies the live blockchain state (balances, return values, permissions, code hashes, timestamps, and more). If any single assertion fails, the entire transaction reverts. No partial execution, and no unexpected outcomes.

  


Capabilities:

  * **Call-based assertions.** Execute any view function on any contract and assert the return value (uint256, address, bool, bytes32, string, raw bytes).
  * **Comparison operators.** Equal, not equal, greater than, less than, greater/less or equal, approximate equality (within delta).
  * **Tuple indexing.** Assert specific return values from functions that return multiple values
  * **Array length assertions.** Validate dynamic array sizes.
  * **Native balance assertions.** Verify ETH balances of any address.
  * **Block & chain assertions.** Assert block number, timestamp, and chain ID.
  * **Contract existence checks.** Verify code presence and exact code hashes (critical for upgrade verification).
  * **Custom error messages.** Every assertion has an overloaded version with descriptive messages for debugging.



  


Where to use it:

  * **Safe Transaction Builder.** Batch assertions as additional transactions in your Safe multisig batch.
  * **DAO proposals.** Embed assertions directly in governance proposal action lists (Aragon, Governor, Compound, etc.).
  * **EIP-7702 batches.** Include assertions in Smart Account call batches for EOA-initiated multi-step operations.



  


## Real-world impact

Assertions was used in [TheDAO Security Fund reactivation](https://paragraph.com/@thedao.fund/thedao-security-fund-activating-75000-eth-for-ethereum-security) to safely recover and distribute over 75,000 ETH (~$220M) in one of the largest on-chain fund movements in Ethereum history. The assertions verified final ETH balances across multiple safes, confirmed DAO token distributions, and validated contract whitelisting, ensuring every step of the [complex multi-sig operation](https://evmcrispr.com/#/terminal/QmVLeRqWmK4PnPdZ6RzYDpLoVWoQf3fN2r6ru1rJRhHaDB) completed exactly as intended.
