## Description

**Vero** is a multi-node validator client designed to protect validators from client bugs. It does this by cross-checking the blockchain state across multiple client implementations before submitting attestations.

  


  


## How does Vero help Ethereum?

Voting for an incorrect chain can have disastrous for Ethereum validators – in the worst case, all of their staked ETH can get slashed. By using Vero, Ethereum validators can avoid voting for an incorrect chain by getting a sort of "second opinion" before they attest.

  


  


## What is Vero's goal?

The short-term goal is to protect as many validators as possible from client bugs.

The long-term goal is to protect all of Ethereum from the potentially catastrophic consequences of client bugs. To reach this goal, we need about 35-40% of the network to run Vero (or equivalent) setups.

  


  


## Is Vero production-ready?

Yes, Vero is production-ready! Vero is already making thousands of validators more resilient to client bugs, across both Ethereum and Gnosis Chain.

  


  


## What is coming up for Vero?

As a validator client, Vero needs to stay up-to-date with the latest network upgrades. We have added support for the Fusaka network upgrade in 2025, and have started working on the upcoming Glamsterdam network upgrade that is expected to ship in 2026.

  


Besides network upgrades we also regularly ship new features and optimizations, see Vero's [changelog](https://vero.docs.serenita.io/reference/changelog/).
