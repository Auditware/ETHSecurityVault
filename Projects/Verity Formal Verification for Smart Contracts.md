## What?

Verity is a formally verified smart contract compiler written in [Lean 4](https://lean-lang.org/). You write contracts in an embedded DSL, state what they should do, prove those properties hold, and compile to EVM bytecode.

  


It can be used in two main ways:

  * Modeling existing Solidity contracts and formally verifying them.
  * Writing new contracts with built-in proofs



It's already possible to use it, and we've worked with the best Ethereum protocols to formally verify production DeFi contracts with Verity:

  * [Lido V3: solvency invariant of Staking Vaults](https://lfglabs.dev/research/lido-vault-solvency)
  * [Safe: linked-list invariants of the owner list](https://lfglabs.dev/research/safe-owner-reachability)



  


The compiler itself is also fully formally verified, not just the contracts it produces. We're co-authoring a research paper on the framework, how it bridges Lean 4 and the EVM, and why we think formal verification is about to become practical at scale. Verity is fully open-source and MIT-licensed. You can read more about the approach here: <https://lfglabs.dev/research/verity>

## Why

**Why Formal Verification**

Smart contract audits are human opinions frozen in time. An auditor reads your code, checks for known patterns, and writes a report, but the report is not a guarantee. Formal verification means writing down the exact rules your contract must follow, then producing a machine-checkable proof that those rules hold for _every_ possible input and execution path within the model. That matters even more when frameworks could ship with insecure defaults that audits tend to miss. Formal verification catches them at the model level.

  


**In Short:** Formal verification gives you a real guarantee: if the assumptions match the deployed system, the proven properties hold for every possible input, no matter who's attacking.

  


**Why It Matters Now**

AI is getting better every day at finding vulnerabilities that humans and simple automated tools miss. Audits won't scale against adversaries that never sleep and improve weekly thanks to AI model advancements when a formal proof doesn't degrade over time. Once a property is proven, it holds permanently, no matter how sophisticated the attacker.

  


**Why Verity**

Most smart contract verification tools rely on bounded checks: they can only say they found no counterexample up to a depth. Verity produces unbounded, machine-checkable proofs. It also tackles two problems the ecosystem has long ignored. First, compiler risk: a proven contract can still be betrayed by an unverified compiler, so Verity’s toolchain is formally verified. Second, language limits: Solidity favors expressiveness over safety, so Verity’s embedded DSL is built for end-to-end guarantees. Verity is [fully open-source](https://github.com/lfglabs-dev/verity/) and MIT-licensed, so anyone can inspect the prover, verify proofs, and build on it to make formal verification a common Ethereum security practice.
