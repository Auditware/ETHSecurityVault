## Description

Plank is a new smart contract language for the EVM. It aims to make writing secure contracts easier and more efficient by pushing more checks into the type system and eliminating the need for inline assembly.

  


## Fixing the Ecosystem's Core Bottleneck

Plank is designed to alleviate the main bottleneck holding back other languages like Solidity & Vyper: the bandwidth of compiler engineers. Currently, any non-trivial extension to the language requires updates in the compiler itself. As a result, even relatively simple features widely requested by users can take years to be implemented, such as a generic ERC712 utility or more optimized storage packing.

  


Drawing inspiration from Zig & Rust, Plank empowers users to "help themselves" by providing a powerful & general purpose set of language primitives. This allows users to implement their own ideas and extensions without requiring deep compiler know-how.

  


## Plank's Killer Feature: Comptime

 _Comptime_ is Plank's main differentiator. It allows smart contract developers to add code that only exists at compile time, enabling expressive static checks, code specialization and other functionality to be added by users without fear of bloating their final application.

  


Comptime enables features that are taken for granted in normal programming languages like Javascript, Python or Rust but are not possible in Solidity/Vyper:

  1. Generic Data Structures: Allows users to define and endlessly reuse data structures with different data types
  2. Custom Static Analysis: From math rounding bugs, to mixing of units, comptime allows developers to define their own static analysis & security checks via the type system, without additional gas costs. No janky extra tooling, no custom scripts, just Plank.
  3. Reusable Generic Optimizations: Today, when devs discover a compiler inefficiency, they are forced to rewrite the same patterns in inline assembly again and again. With Plank, such optimizations can be wrapped in reusable generic functions and shared as libraries, avoiding wasting time reauditing the same 10 assembly patterns.
