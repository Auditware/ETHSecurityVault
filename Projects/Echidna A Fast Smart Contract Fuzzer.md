## Description

[Echidna](https://github.com/crytic/echidna/) is a Haskell program designed for fuzzing/property-based testing of Ethereum smart contracts. It uses sophisticated grammar-based fuzzing campaigns based on a contract ABI to falsify user-defined predicates or [Solidity assertions](https://docs.soliditylang.org/en/develop/control-structures.html#error-handling-assert-require-revert-and-exceptions).

  


  


Echidna was created 8 years ago and it was already used in a number of complex real-world fuzzing campaign, most of them are detailed here: <https://github.com/perimetersec/public-fuzzing-campaigns-list>.

##   


## Features

  


  * Generates inputs tailored to your actual code
  * Corpus collection, mutation and coverage guidance to find deeper bugs
  * Powered by [Slither](https://github.com/crytic/slither) to extract useful information before the fuzzing campaign
  * Source code integration to identify which lines are covered after the fuzzing campaign
  * Interactive terminal UI, text-only or JSON output
  * Automatic test case minimization for quick triage
  * Seamless integration into the development workflow using [Github actions](https://github.com/crytic/echidna-action).
  * Optimization mode: instead of break a binary property, which was [also adopted by other fuzzers](https://github.com/foundry-rs/foundry/pull/13196) such as Medusa and Foundry.
  * Efficient on-chain fetching of state to test live contracts and source code fetching (using either [Sourcify](https://sourcify.dev/) or [Etherscan](https://etherscan.io/))



  


## New features

  


In recent releases, Echidna gained a few more advanced features!

  


  * Automatic **Foundry unit-test generation** from each invariant/property failure, for quick debugging
  * Support of **a large number of Foundry cheatcodes,** provided by [hevm](https://github.com/argotorg/hevm)
  * Optional [symbolic execution](https://gustavo-grieco.github.io/blog/echidna-symexec/) for **formal verification or exploration or finding counter-examples** using [hevm](https://github.com/argotorg/hevm)



  


  


## Upcoming agentic capabilities

Echidna will have a [fully integrated MCP](https://github.com/crytic/echidna/pull/1502) that**allows AI agents to inspect and direct fuzzing campaigns while they run.** Agents such as Claude or Codex can be used to explore quickly or try to break specific invariants, using the [preview build already available](https://github.com/crytic/echidna/releases/tag/v2.3.2-agents-preview-1). This feature needs a good amount of experimentation and development to make sure agents can use it efficiently. **This is one key features that will develop faster with the donations received in this round**.
