## Description

## **Hevm is a specialized symbolic execution engine and testing framework for Ethereum smart contracts, serving as a cornerstone tool for smart contract security.  **

Originally developed under the Ethereum Foundation, hevm has found a new home at Argot (argot.org), an independent non-profit organization dedicated to maintaining and developing open source infrastructure for the Ethereum ecosystem.

  


By enabling developers to perform rigorous analysis of contract behavior, uncover edge cases, and verify code properties through symbolic manipulation, **hevm plays an essential role in ensuring the correctness and safety of smart contracts.  **

  


This essentially means that hevm can try out all execution possibilities of your contract and see it can somehow violate some assertions you have. These assertions can be e.g. the total number of tokens must always be X, some value must never be greater than Y, some value must never overflow, etc.

  


In some sense, hevm is similar to a fuzzer, but instead of only trying with random values to trigger faults, it instead computes whether a fault is possible. If it is possible, it gives an example call to trigger the fault, and if it isn't possible, it mathematically proves it so, and tells the user the contract is safe.

  


Hevm can not only be used to find bugs in programs, but can also help to make sure that two programs behave equivalently from the outside. This may be advantageous when one may be more efficient (use less gas) to execute, but harder to reason about. This can be done via equivalence checking where hevm either proves that the behaviour of the two bytecodes is the same, or gives inputs where they differ.

  


## Recent work:

Recent work has centered on hardening the tool through maintenance-focused releases, delivering meaningful improvements to performance, reliability, and usability. Already powering tools like [Echidna](https://github.com/crytic/echidna/), with symbolic execution now progressively integrated into both Echidna and the [Solidity](https://www.soliditylang.org/) fuzzing pipeline, hevm is increasingly being positioned as a foundational execution engine at the core of the broader verification and testing ecosystem, with the team's focus going forward shifting toward deepening that role and enabling more tools to build on top of it.

## How the donations will be used:

100% of donated funds go directly to engineer salaries. hevm is public good infrastructure, maintained by a small team at Argot, a non-profit organization. Contributions fund the work needed to ship a browser-based version of hevm via WebAssembly, advance stateful invariant testing, and continue the deeper integration with Echidna that combines fuzzing and symbolic execution for broader path discovery. Every dollar donated goes toward keeping this critical, open-source tooling maintained and moving forward. See full roadmap [here](https://www.argot.org/blog/2026-01-15-argot-roadmap-update-2026-1#hevm).
