## Description

**EVM Chronicle** is an EVM contract state research tool, currently focused on **Ethereum** , that helps people inspect real storage, mapping state, historical variable changes, execution flow, and code patterns, with the long-term goal of becoming an **EVM search engine**.

##   


## EVM Chronicle 4 Main Functions

**1\. Storage Explorer**

Inspects real Ethereum storage, including slots, offsets, mappings, structs, arrays, and value history. It helps users see how contract state is actually stored and how it changes over time, especially for mapping variables that are usually difficult to inspect.

  


**2\. Transaction Tracer**

Tracks execution flow, including variables read and variables changed. It helps users understand what a transaction actually did at the storage level, not just which logs were emitted or which functions were called.

  


**3\. Source Code Intelligence**

Lets users inspect storage directly from source code and simulate functions in the same workflow. It helps users move from source to state to execution without needing separate tools or a local setup.

  


**4\. Contract Search**

Searches contracts by string matching and code patterns. It helps users find contracts that contain specific code, logic structures, or vulnerability-related patterns, which is useful for research, auditing, and bug hunting.

## **Why it matters**

Even today, understanding EVM storage values is still difficult, time consuming, and too manual, especially when it comes to mapping variables. My hope is that EVM Chronicle can reduce that gap, including for people without Solidity knowledge who still need to extract contract data and do meaningful analysis.

  


  


## **Next Milestones**

**1\. Index from genesis and maintain realtime indexing**

Expand EVM Chronicle from range-based indexing to full indexing from genesis to the latest block, and eventually maintain realtime indexing as each new block is produced.

  


**2\. Adopt Ethereum Layer 2s**

Extend EVM Chronicle beyond Ethereum mainnet and bring the same storage, state, and execution visibility to Layer 2 networks.

  


**3\. Become an EVM search engine**

Build EVM Chronicle toward an EVM search engine, where someone can search for a specific value, variable, mapping key, or code and immediately discover where it exists in a specific contract.
