## Description

Etherform is an open-source security and DevSecOps toolkit of reusable GitHub Actions workflows for Foundry smart contract CI/CD. It is designed to help smart contract teams standardize and automate core development and deployment tasks through reusable workflows instead of rebuilding the same CI/CD setup in each repository.

The project includes workflows for build and test pipelines, formatting checks, coverage, Halmos runs, testnet deployment, Blockscout verification, and upgrade safety validation. A core part of Etherform is upgrade safety for proxy-based contracts using the OpenZeppelin upgrades-core CLI. This validation checks storage layout compatibility, initializer safety, and proxy semantics. The upgrade-safety workflow compares contracts against the base branch or against an explicitly defined reference contract.

In addition to the individual workflows, the project includes an all-in-one workflow that combines CI, upgrade validation, and testnet deployment into a single orchestrated pipeline.

Overall, Etherform is focused on making Foundry-based smart contract CI/CD more reusable, configurable, and safety-oriented for Ethereum development workflows across teams and repositories.
