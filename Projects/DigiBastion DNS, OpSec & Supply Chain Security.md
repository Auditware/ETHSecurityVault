## Description

[DigiBastion](https://www.digibastion.com/) is an open-source Web3 security project founded by [Raiders](https://x.com/__Raiders) ([web3sec.news](https://web3sec.news/) founder), an [Ethereum Foundation grantee,](https://blog.ethereum.org/2025/12/02/allocation-q3-25) [SEAL Alliance contributor,](https://securityalliance.org/members/user_DtuOuJvlEHb4ozUK) and published [MITRE AADAPT](https://aadapt.mitre.org/techniques/ADT1195) researcher. Built for Web3 protocols, developers, and users who need defense that goes far beyond smart contract audits.

  


DigiBastion is a full-stack defensive security platform built ground-up for the Web3 threat model. It covers:

  * DNS and domain security scanning across Ethereum mainnet and L2 protocols, currently in closed beta with direct outreach to protocol founders, developers, and heads of security (BETA)
  * Supply chain and JavaScript integrity monitoring for protocol frontends (WIP)
  * Phishing detection and active lookalike domain tracking (WIP)
  * Domain risk scoring browser extension for users (WIP)
  * Real-time threat intelligence feeds for protocols, researchers, and incident responders (LIVE)
  * Operational security guides, checklists, and quizzes for developers and everyday users (LIVE)



  


The **DNS security scanner** is currently operating in a structured **closed beta**. Rather than dumping findings publicly, we reach out directly to protocol teams, share findings privately, work with them to fix issues, and collect feedback before the tool goes fully public. This responsible disclosure approach ensures real fixes happen before attackers notice the same gaps.

  


Unlike generic security tools retrofitted for crypto, every component of DigiBastion was designed specifically for the attack surfaces draining Web3 users today: lookalike domains, DNS hijacks, malicious frontends, compromised dependencies, and supply chain poisoning.

  


The full platform is open source and publicly accessible at digibastion.com. The codebase is available on GitHub at <https://github.com/Raiders0786/digibastion>

  


**Why?**

Most losses in Web3 today don't come from novel smart contract exploits. They come from phishing, DNS hijacks, supply chain compromises, and malicious frontends. Yet the ecosystem remains heavily over-indexed on audits and under-invested in the continuous infrastructure-layer defense that actually protects users in real time.

  


Audits are snapshots. Attacks are continuous.

  


DigiBastion fills that gap as a public good. Through the closed beta responsible disclosure program, scanning top Ethereum and L2 protocols has already surfaced critical live findings shared directly with protocol teams:

  * Axelar: score 96
  * Lido: score 91
  * Chainlink: score 90
  * GMX: score 93
  * QuickSwap: score 90



  


The [real-time threat intelligence feeds](https://www.digibastion.com/threat-intel) also serve a direct incident response function. When a DNS hijack or frontend compromise is suspected, responders need live data on domain status, lookalike activity, and infrastructure posture. DigiBastion provides exactly that, continuously updated and freely accessible.

  


Foundation work includes authoring the SEAL Alliance Domain and DNS Security Framework ([frameworks.securityalliance.dev/infrastructure/domain-and-dns-security](https://frameworks.securityalliance.dev/infrastructure/domain-and-dns-security/)) and publishing supply chain attack TTPs to MITRE AADAPT ([aadapt.mitre.org/techniques/ADT1195](https://aadapt.mitre.org/techniques/ADT1195/)), establishing the research and standards credibility behind the tooling.

  


## **How?**

DigiBastion runs as a continuously maintained open-source project. Current coverage includes 50+ top Ethereum and L2 protocols scanned regularly, threat intel feeds updated in real time, and a growing library of opsec resources used by developers and users across the ecosystem.

  


Running this at the level the ecosystem actually needs isn't cheap. Unlike a one-time audit or a static report, DigiBastion operates as live infrastructure across multiple interconnected systems that all require continuous funding to stay operational.

  


**Infrastructure and Services:**

  * DNS and domain scanning infrastructure running daily across 50+ protocols
  * Supply chain and JavaScript integrity monitoring with automated diff detection
  * Real-time threat intel scraping, aggregation, and feed generation
  * Database hosting and storage for historical findings, domain records, and threat data
  * API layer for researcher and incident responder access
  * Browser extension backend and update pipeline
  * Uptime monitoring, alerting, and redundancy across services



  


**Research and Development:**

  * Ongoing protocol coverage expansion across Ethereum L2s
  * New detection modules for emerging attack vectors including BGP hijacks, RPC poisoning, and frontend injection
  * Building a shared open DNS security data standard for the broader ecosystem
  * Integration work with existing security tooling and incident response workflows
  * Closed beta program management, responsible disclosure coordination, and protocol team engagement



  


**Human Cost:**

  * Continuous research, triage, and validation of findings
  * Threat intel curation and publishing
  * Community education content via Web3Sec News distributed to 10,000+ readers
  * Coordination with SEAL Alliance, MITRE, and ecosystem partners

This is not a one-time build. It is continuous defense infrastructure.
