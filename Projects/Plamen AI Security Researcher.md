## Description

Web3 depends on open infrastructure, but security is still too expensive, too slow, and too concentrated. Many public-good projects cannot afford deep security review, and many users only discover weaknesses after funds, governance systems, or communities are already at risk.

  


Plamen exists to change that.

  


Plamen is an open security-audit system that helps find vulnerabilities in blockchain projects before attackers do. It combines automated analysis, specialist security agents, verification steps, and human-readable reporting to make high-quality security review more accessible to the projects that need it most.

  


The goal is simple: help public-good protocols, infrastructure teams, and open-source builders ship safer systems without wasting resources on a manual audit that will waste time on the same classes of bugs that LLMs can find and will instead help the manual review focus on the deep research.

  


## Why This Matters

Security failures in crypto do not only hurt teams. They hurt users, donors, DAOs, public-good funding pools, and the broader trust in open financial infrastructure.

  


A single missed bug can lead to stolen funds, halted networks, broken governance, or loss of confidence in projects doing valuable work. At the same time, many public-good teams are priced out of traditional audits, especially early in their lifecycle.

Plamen focuses on closing that gap by building an audit pipeline that can repeatedly inspect large codebases, search for known and novel bug patterns, verify findings, and produce reports that developers can act on.

This is not meant to replace expert auditors. It is meant to give more projects a strong first line of defense, reduce obvious risk before expensive review, and help human auditors spend more time on the hardest problems.

Donations help improve and operate Plamen as a public-good security tool.

  


Funding will support the development of:

 \- Better vulnerability detection across smart contracts, L1 clients, bridges, DeFi protocols, and Move-based

  ecosystems.

 \- More reliable verification so real bugs are preserved and false positives are filtered responsibly.

 \- Better reports that developers can understand and fix quickly.

 \- Open methodology improvements based on real audit misses, postmortems, and public vulnerability research.

 \- Testing Plamen against large real-world codebases so the system improves from practical evidence, not theory.

 \- Making security knowledge reusable across audits instead of locked inside one-off private engagements.

  


## Impact

  


 A donation to Plamen helps create more accessible security coverage for the Web3 ecosystem.

  


 With more support, it could:

 \- Run more benchmark audits against real protocols.

 \- Improve recall on critical vulnerability classes.

 \- Reduce false positives and noisy reports.

 \- Add methodology for new ecosystems like Sui Move.

 \- Publish clearer guidance for projects and auditors.

 \- Help open-source teams identify dangerous issues earlier.

  


Even small improvements in security tooling can have large impact. If Plamen helps one project catch a critical bug before launch, the value protected may be far greater than the cost of building the tool.

  


## Milestones

  


 Our near-term milestones are:

  


 1\. Stabilize the audit pipeline so findings are not lost between discovery, verification, and reporting.

 2\. Improve report quality so developers receive clear, actionable findings instead of noisy technical dumps.

 3\. Expand coverage for recurring missed bug classes across L1 clients, smart contracts, and Move projects.

 4\. Benchmark against known ground-truth audits and publish what improved, what failed, and what still needs work.

 5\. Make the system easier for public-good projects to use as a pre-audit or continuous security review layer.
