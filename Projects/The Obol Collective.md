## Description

Obol builds **Distributed Validator Technology (DVT) for Ethereum**. Our flagship software, Charon, is a middleware client that enables **any Ethereum validator to run across multiple independent nodes instead of relying on a single machine**. It works by sitting between the consensus client and validator client, coordinating threshold signing across a cluster of nodes - so a validator can tolerate node failures, network partitions, and even compromised operators without going offline or getting slashed.

  


Charon clusters use Distributed Key Generation so the validator's private key never exists in one place - a threshold of independent nodes (e.g., 3-of-4) must agree to sign, eliminating single points of failure. This active-active redundancy replaces fragile failover setups that are the leading cause of double-signing slashing events on Ethereum. Because Charon is middleware, each node in a cluster can run different execution and consensus clients, reducing correlated failure risk from client bugs and improving Ethereum’s client diversity. And by distributing trust across multiple independent operators, DVT directly mitigates the concentration risk that forced the emergency exit of over 1.6M ETH from a single compromised provider in September 2025.

  


Obol currently**secures billions in staked ETH across mainnet** , trusted by protocols including Lido, EtherFi, and Liquid Collective, and more, as well as operators and traditional financial institutions including Blockdaemon, Bitcoin Suisse, New Zealand ETPs, and the even state of Lichtenstein. Obol actively decentralises and thus secures Ethereum with this technology.
