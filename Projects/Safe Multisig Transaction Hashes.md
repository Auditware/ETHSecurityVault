## Description

## Who?

Anyone who doesn't want to get rekt signing a multisig transaction. This script is used across the Ethereum ecosystem - from small teams to blue-chip projects - to **locally verify and simulate Safe transactions** before signing.

  


## What?

A **powerful, local-first Bash CLI tool** (just one file guys!) that strengthens the security and transparency of any Safe multisig operations (transactions and off-chain messages are supported). It lets you independently verify and simulate transactions **before** signing them. If Bybit had used it, DPRK would be about $1.5 billion poorer.

  


## Why?

Because verifying what's shown on your hardware wallet is **what keeps you safe**. If you skip that step, you're trusting someone else's data, and guess what, that's how you get rekt (see Bybit or Radiant Capital).

  


## Where?

Runs on any OS with Bash, or via Docker/Podman. You can execute it entirely **locally** with the right parameters and verify exactly what you're signing.

  


## How?

The script **reconstructs Safe transaction hashes** by fetching transaction details from the Safe Transaction Service API and computing both domain and message hashes according to the EIP-712 standard. Using the --interactive mode, you can work completely offline - entering parameters manually and verifying hashes without relying on any external API.

  


**Core capabilities:**

  * Reproduces the exact transaction/message hashes computed by Safe smart contracts.
  * Supports all Safe multisig versions starting from 0.1.0.
  * Works across 35+ networks.
  * Supports manual parameter entry for standalone or unlogged transactions.
  * Supports nested Safes (computes both primary and nested Safe approval hashes).
  * Generates Safe-compatible EIP-712 message hashes from any input (off-chain messages).
  * Simulates locally the Safe transaction**** to produce execution traces and detect runtime issues before signing.
  * Warns about untrusted delegate calls, owner/threshold changes, and suspicious gas or refund settings etc.
  * Performs all cryptographic computations locally; only simulations require an RPC provider.



  


## When?

Whenever you sign a **Safe multisig transaction** or **message**.
