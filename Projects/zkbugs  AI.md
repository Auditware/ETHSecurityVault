## Description

zkSecurity is building open-source public-good infrastructure for Ethereum and L2 security through better security for zero-knowledge circuits, proving systems, and tooling used across the Ethereum ecosystem. We maintain zkbugs**(**[https://github.com/zksecurity/zkbugs/](https://github.com/zksecurity/zkbugs)**)** , a public dataset of real ZK vulnerabilities, and will expand it from ~110 to 200 bugs, starting with Circom and Plonky3. We will also create an open ZK security skills repository for auditing ZK circuits and proving systems, inspired by pashov/skills (<https://github.com/pashov/skills>), to help developers, auditors, researchers, and AI tools understand bug patterns, reproduce issues, and use the right tools. Finally, we will build a public benchmark for ZK tools and AI auditors, including evaluations of frontier models with and without the skills layer. This builds on our active, verifiable work in Ethereum security and will remain openly accessible, with public progress updates and a post-round report. Funding will support bug curation, reproducibility, skills creation, and benchmark infrastructure.

**Use of Funds**

  


  * **< $5,000:** expand and curate the zkbugs dataset by identifying, validating, and documenting additional real-world ZK vulnerabilities, improving metadata quality and coverage across Ethereum- and L2-relevant ZK ecosystems.
  * **$5,000–$10,000:** reproduce a first set of Circom bugs, creating working proof-of-concept environments and public documentation so developers and auditors can understand and learn from real vulnerabilities.
  * **$10,000–$25,000:** reproduce as many Circom and Plonky3 bugs as possible, turning zkbugs into a stronger public benchmark with reliable, reusable reproduction setups across two important ZK DSLs/frameworks.
  * **$25,000–$50,000:** build and open-source a ZK security skills repository for auditing ZK circuits and proving systems, and develop ZKBugs Arena infrastructure: a benchmark environment containing both real historical bugs and intentionally introduced bugs in otherwise secure code, to reduce bias and improve evaluation quality.
  * **> $50,000:** run continuous, multi-turn evaluations of frontier and open-source AI models on ZK security tasks using zkbugs, the skills repository, and ZKBugs Arena, publishing results and maintaining the benchmark for as long as funding allows.
